# Progress Report Template

Use this template when the Planning Lead role reports milestone progress to the user. This is a direct user-facing message template, not an internal record.

## Usage Principles

- Generate the final user-facing message in the Documentation Language defined in `rules/language-policy.md` or the project's `project-policies.md`.
- Lead with what the user cares about: what got done, what changed, what's next.
- Do not lead with internal identifiers, file names, or technical details.
- Keep it concise. If a milestone is on track, a short report is better than a long one.

## Template

### 1. Current Milestone Status

- Milestone name (in user-facing terms, not internal labels)
- Overall status: on track / delayed / blocked / completed
- What was accomplished since the last report

### 2. Items Not Completed

- What was planned but not completed, and why
- Blockers that prevented completion

### 3. Risks and Debt

- New risks or technical debt discovered during the work
- Items from the previous report that are still unresolved

### 4. Deviations

- Changes from the original plan or assumptions
- Decisions that were made autonomously (with rationale)

### 5. Next Steps

- What is planned for the next cycle
- Any decisions needed from the user before proceeding
